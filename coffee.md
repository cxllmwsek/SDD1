
# Software Requirements Specification (SRS)
## ระบบการจัดการร้านกาแฟพร้อมระบบออเดอร์ออนไลน์
**Version 1.0**

---

## 1. บทนำ (Introduction)

### 1.1 วัตถุประสงค์ (Purpose)
เอกสารฉบับนี้มีวัตถุประสงค์เพื่อระบุข้อกำหนดความต้องการของซอฟต์แวร์สำหรับ  
**ระบบจัดการร้านกาแฟพร้อมระบบออเดอร์ออนไลน์** อย่างครบถ้วน ชัดเจน และเป็นมาตรฐาน  
เพื่อใช้เป็นเอกสารอ้างอิงสำหรับ

- นักพัฒนาซอฟต์แวร์  
- นักออกแบบระบบ  
- ผู้ทดสอบระบบ  
- ผู้บริหารโครงการ  
- ผู้มีส่วนได้ส่วนเสีย (Stakeholders)

เอกสารนี้ครอบคลุมทั้งความต้องการด้านฟังก์ชันและไม่ใช่ฟังก์ชัน

### 1.2 ขอบเขตของระบบ (Scope)
ระบบจัดการร้านกาแฟรองรับการทำงานทั้งหน้าร้านและออนไลน์ โดยมีความสามารถหลักดังนี้

- รับออเดอร์และชำระเงิน  
- บริหารจัดการเมนู ราคา และโปรโมชั่น  
- บริหารจัดการสต็อกวัตถุดิบ  
- จัดการสมาชิกและแต้มสะสม  
- ติดตามสถานะออเดอร์แบบเรียลไทม์  
- วิเคราะห์ข้อมูลการขายและต้นทุน  
- รองรับการทำงานแบบออฟไลน์บางส่วน  

### 1.3 คำจำกัดความ คำย่อ และตัวย่อ

| คำศัพท์ | ความหมาย |
|-------|---------|
| POS | Point of Sale |
| SRS | Software Requirements Specification |
| OTP | One-Time Password |
| PDPA | พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล |
| SKU | Stock Keeping Unit |

### 1.4 ภาพรวมเอกสาร
- บทที่ 2: ภาพรวมของระบบ  
- บทที่ 3: ความต้องการด้านฟังก์ชัน  
- บทที่ 4: ความต้องการที่ไม่ใช่ฟังก์ชัน  
- บทที่ 5: ข้อจำกัดและสมมติฐาน  
- บทที่ 6: ภาคผนวกและการอนุมัติ  

---

## 2. ภาพรวมของระบบ (Overall Description)

### 2.1 มุมมองผลิตภัณฑ์
ระบบเป็น Web-based และ Mobile Application เชื่อมต่อฐานข้อมูลกลาง  
ประกอบด้วยโมดูล

- POS Frontend  
- Customer Mobile Application  
- Admin Dashboard  
- Inventory Management System  
- Payment Gateway Integration  

### 2.2 กลุ่มผู้ใช้งาน

| ผู้ใช้งาน | คำอธิบาย |
|---------|---------|
| ลูกค้า | สั่งสินค้า ติดตามออเดอร์ ใช้แต้ม |
| พนักงานหน้าร้าน | รับออเดอร์ ทำเครื่องดื่ม |
| ผู้จัดการร้าน | จัดการเมนู สต็อก รายงาน |
| ผู้ดูแลระบบ | ดูแลสิทธิ์และความปลอดภัย |

### 2.3 สมมติฐานและการพึ่งพา
- ระบบออนไลน์ต้องใช้อินเทอร์เน็ต  
- ระบบชำระเงินพึ่งพาผู้ให้บริการภายนอก  
- POS สามารถ Sync ข้อมูลภายหลัง

## 3. ความต้องการด้านฟังก์ชัน (Functional Requirements)

> รูปแบบรหัส FR-XXX

### 3.1 ระบบหน้าร้าน (POS)

#### การจัดการเมนู
- **FR-POS-01** เพิ่ม / แก้ไข / ลบ เมนู  
- **FR-POS-02** รองรับตัวเลือกเครื่องดื่ม  
- **FR-POS-03** กำหนดราคาตามประเภทลูกค้า  

#### การรับออเดอร์
- **FR-POS-04** รับออเดอร์ Walk-in  
- **FR-POS-05** คำนวณราคารวมอัตโนมัติ  
- **FR-POS-06** พิมพ์สลิปและใบเสร็จ  

#### การจัดการคิว
- **FR-POS-07** แสดงลำดับคิว  
- **FR-POS-08** อัปเดตสถานะแบบเรียลไทม์  

### 3.2 ระบบออเดอร์ออนไลน์

#### แอปลูกค้า
- **FR-APP-01** สมัครและเข้าสู่ระบบ  
- **FR-APP-02** แสดงเมนูพร้อมรูป  
- **FR-APP-03** ตะกร้าสินค้า  
- **FR-APP-04** ติดตามสถานะออเดอร์  

#### การจัดการออเดอร์
- **FR-ONLINE-01** แจ้งเตือนออเดอร์ใหม่  
- **FR-ONLINE-02** จัดลำดับออเดอร์อัตโนมัติ  
- **FR-ONLINE-03** ยกเลิกออเดอร์ตามเงื่อนไข  

### 3.3 ระบบสต็อก
- **FR-INV-01** ตัดสต็อกอัตโนมัติ  
- **FR-INV-02** แจ้งเตือนวัตถุดิบใกล้หมด  
- **FR-INV-03** บันทึกวันหมดอายุ  

### 3.4 ระบบชำระเงิน
- **FR-PAY-01** เงินสด / บัตร / QR  
- **FR-PAY-02** ตรวจสอบสถานะอัตโนมัติ  
- **FR-PAY-03** ออกใบเสร็จและใบกำกับภาษี  

### 3.5 ระบบสมาชิก
- **FR-MEM-01** คำนวณแต้มสะสม  
- **FR-MEM-02** ระดับสมาชิก  
- **FR-MEM-03** แลกของรางวัล  

### 3.6 Dashboard และรายงาน
- **FR-REP-01** รายงานยอดขาย  
- **FR-REP-02** รายงานสต็อกและต้นทุน  
- **FR-REP-03** Export รายงาน

## 4. ความต้องการที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)

### 4.1 Performance
- **NFR-PER-01** Response ≤ 2 วินาที  
- **NFR-PER-02** รองรับ ≥ 1,000 ผู้ใช้  
- **NFR-PER-03** POS ทำงาน Offline  

### 4.2 Security
- **NFR-SEC-01** เข้ารหัสข้อมูล  
- **NFR-SEC-02** รองรับ OTP  
- **NFR-SEC-03** ปฏิบัติตาม PDPA  

### 4.3 Usability
- ภาษาไทย  
- รองรับทุกหน้าจอ  
- มีคู่มือใช้งาน  

### 4.4 Reliability
- Uptime ≥ 99.5%  
- Backup อัตโนมัติ

## 5. ข้อจำกัดของระบบ
- iOS 12+ / Android 8+  
- ต้องใช้อินเทอร์เน็ตสำหรับออนไลน์  
- เก็บข้อมูลย้อนหลังไม่เกิน 1 ปี
